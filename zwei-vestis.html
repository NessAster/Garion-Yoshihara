Zwei Vestis

Dans un cadre personnel, je réalise une application sur la plateforme de discussion Discord. Cette application permet de gérer l'administration de serveurs communautaires, de gérer un système d'inventaire pour les utilisateurs, ainsi qu'un système de combat basé sur un lancer de pièce pour un jeu de rôle.

J'ai commencé à réaliser ce projet au début de mes années de BUT, mais mon code était très vite peu maintenable. Donc, quand j'ai acquis plus d'expérience, j'ai repris l'application de zéro afin de repartir sur de bonnes bases. Reprendre cette application de zéro m'a permis d'optimiser le code que j'avais déjà réalisé. J'ai pu mieux le découper afin qu'il soit plus maintenable et propre.

J'ai réalisé cette application en Python grâce à la bibliothèque API Discord.py, qui permet de récupérer les événements qui se déroulent sur la plateforme Discord. Bien que je n'étais pas assez expérimenté lorsque j'ai développé cette application, cela m'a permis de prendre conscience de l'importance d'un code maintenable et bien structuré. Étant seul sur le projet, j'ai pu développer mon autonomie en cherchant dans la documentation à disposition, qui n'était pas forcément très claire, afin de réaliser les fonctionnalités voulues.

Ayant créé un jeu de rôle avec son propre système basé sur des lancers de pièces et non des lancers de dés, contrairement aux autres jeux de rôle, il me fallait donc une application pour réaliser ces lancers spécifiques. À l'heure actuelle, cette application sert notamment pour ce jeu de rôle en automatisant le lancer des jets de compétences et tout ce qui est lié aux compétences des joueurs, leur permettant ainsi de simplifier au maximum ces derniers. Cela m'a demandé une procédure récursive pour le traitement des compétences, puisqu'elles impliquent des effets pouvant entraîner d'autres effets, ce qui m'a demandé de réfléchir de façon récursive et de penser à toutes les subtilités des règles que j'avais établies à la base sans les envisager d'un point de vue algorithmique. Cependant, en découpant bien mes fonctionnalités afin qu'elles s'occupent chacune d'une partie propre, cela a rendu la récursion plus simple. Ainsi, les joueurs peuvent effectuer leurs jets sans faire d'erreur ou sans avoir à se rappeler exactement de la commande à utiliser. De plus, j'ai été amené à réaliser des fonctionnalités temporaires, telles qu'un calendrier de l'avent pour les périodes d'été et pour des événements, avec un système de validation par compte pour que les utilisateurs ne puissent utiliser ces fonctionnalités qu'une seule fois.

Ce projet m'a permis d'approfondir l'architecture de mon code et de gagner en autonomie en réalisant cette application seul. J'ai appris à mieux m'organiser et à conceptualiser mes fonctions afin de les rendre les plus efficaces et modulables possible.
